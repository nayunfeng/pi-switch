# Pi Switch Wireframes

## 1. 线框结论

第一版使用紧凑桌面工具布局：

```text
顶部栏
左侧 Profile 列表
右侧 Profile 表单
底部测试输出区
```

不做 landing page，不做大卡片式营销布局，不做配置 JSON 预览。

布局规则：

- 左侧宽度固定或最小 260px。
- 右侧表单占剩余空间。

- 按钮使用图标 + 短文本。
- 删除按钮使用明确危险色，但不要过度醒目。

- 默认模型通过模型行上的 default 控件设置。

- API type 用下拉预设 + Custom 输入。
- 模型 ID 原样保存，不 trim。

## 6. 拉取模型候选列表

成功拉取：

```text
┌──────────────────────────────────────────┐
│ Fetched models                           │
│ [ Search models...                    ]  │
│                                          │
│ [ ] deepseek-chat                        │
│ [x] deepseek-coder                       │
│ [ ] gpt-4o                               │
│ [x] claude-sonnet-4                      │
│                                          │
│ [ Add selected ] [ Close ]               │
└──────────────────────────────────────────┘
```

失败：

```text
┌──────────────────────────────────────────┐
│ Fetch failed                             │
│ stderr / error text                      │
│                                          │
│ [ Close ]                                │
└──────────────────────────────────────────┘
```

规则：

- 候选模型不自动加入 Profile。
- 用户勾选后加入。
- 失败后仍可手动添加模型 ID。

## 7. 删除确认

```text
┌──────────────────────────────────────────┐
│ Delete Profile                           │
│                                          │
│ 确定删除 Profile "Relay Daily"？         │
│                                          │
│ [ Cancel ] [ Delete ]                    │
└──────────────────────────────────────────┘
```

规则：

- Delete 是危险操作样式。
- 确认后删除 GUI 配置里的 Profile。
- 不清理 `auth.json`。

## 8. 应用失败状态

```text
┌─────────────────────────────────────────────────────────────────────┐
│ Apply failed                                                        │
│                                                                     │
│ Failed file: settings.json                                          │
│ Written files: models.json, auth.json                               │
│                                                                     │
│ 修正问题后可重新点击“应用到 Pi”。                                  │
└─────────────────────────────────────────────────────────────────────┘
```

规则：

- 应用成功不显示。
- 应用失败必须显示失败文件和已写成功文件。
- 不做回滚入口。


- 输出不保存。
- 输出不脱敏。
- 可限制区域高度并滚动。

## 10. 字段级错误

Provider ID 冲突：

```text
Provider ID
[ openai                                                       ]
该 provider ID 已用于内置供应商。
```

API Key 为空：

```text
API Key
[                                                             ]
API Key 不能为空。
```

默认模型不存在：

```text
Models
默认模型必须存在于模型列表中。
```

## 13. 待验证点

低保真原型后仍需验证：

- 模型列表在数量多时是否占用太多空间。
- 测试输出区域放底部还是右侧表单内更顺手。
- 成功不提示是否会导致用户重复点击。
- 自定义 provider 表单是否过长，需要分组或折叠。
- API Key 默认脱敏但可明文是否足够顺手。
